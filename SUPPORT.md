# Support

Thanks for using TokenHelm! Here's where to go for help.

## Questions & usage help

- **GitHub Discussions** — https://github.com/quadkeys/tokenhelm/discussions
  Best place for "how do I…", design questions, and sharing patterns.
- **Documentation** — start with [`docs/quickstart.md`](docs/quickstart.md),
  [`docs/configuration.md`](docs/configuration.md), and
  [`docs/api-reference.md`](docs/api-reference.md).

## Bugs & feature requests

- Open an issue using the appropriate template:
  [bug report](.github/ISSUE_TEMPLATE/bug_report.md) or
  [feature request](.github/ISSUE_TEMPLATE/feature_request.md).
- Please include version, Python, OS, and a minimal reproduction.

## Security issues

Do **not** open a public issue. Follow [`SECURITY.md`](SECURITY.md).

## Before asking

1. Confirm your version: `python -c "import tokenhelm; print(tokenhelm.__version__)"`.
2. Search existing issues and discussions.
3. Try to reduce the problem to a minimal example (with credentials removed).

## Response expectations

TokenHelm is maintained on a best-effort basis. Issues are triaged regularly, but response
times vary. Well-scoped reports and PRs get attention fastest.
